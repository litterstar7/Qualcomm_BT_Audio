Detailed Description

Header file for the GATT battery client library.

This file provides documentation for the GATT battery client library API (library name: gatt_battery_client).

Typedef Documentation

◆ GBASC

typedef struct __GBASC GBASC

The battery internal structure for the client role.

This structure is visible to the application as it is responsible for managing resource to pass to the battery library. The elements of this structure are only modified by the battery library. The application SHOULD NOT modify the values at any time as it could lead to undefined behaviour.

◆ GattBatteryClientDestroy()

bool GattBatteryClientDestroy ( GBASC *  battery_client)

When a GATT connection is removed, the application must remove all client service instances that were associated with the connection (using the CID value). This is the clean up routine as a result of calling the GattBatteryClientInit API. That is, the GattBatteryClientInit API is called when a connection is made, and the GattBatteryClientDestroy is called when the connection is removed.

Parameters
battery_clientThe client instance that was passed into the GattBatteryClientInit API.
Returns
TRUE if successful, FALSE otherwise

◆ GattBatteryClientInit()

bool GattBatteryClientInit ( GBASC *  battery_client,
Task  app_task,
uint16  cid,
uint16  start_handle,
uint16  end_handle,
gatt_battery_client_device_data_t *  device_data 
)

After the VM application has used the GATT manager library to establish a connection to a discovered BLE device in the Client role, it can discover any supported services in which it has an interest. It should then register with the relevant client service library (passing the relevant CID and handles to the service). For the battery client it will use this API. The GATT manager will then route notifications and indications to the correct instance of the client service library for the CID.

Parameters
battery_clientA valid area of memory that the service library can use.
app_taskThe Task that will receive the messages sent from this battery client library.
cidThe connection ID.
start_handleThe start handle of the battery client instance.
end_handleThe end handle of the battery client instance.
device_dataPersistent data for the device.
Returns
TRUE if successful, FALSE otherwise

◆ GattBatteryClientReadDescriptorRequest()

void GattBatteryClientReadDescriptorRequest ( const GBASC *  battery_client,
uint16  descriptor_uuid 
)

This API is used to read a descriptor of the battery level characteristic. This will only read the first descriptor found with the specified UUID, as it assumes a unique characteristic descriptor UUID exists per battery level characteristic.

Parameters
battery_clientThe client instance that was passed into the GattBatteryClientInit API.
descriptor_uuidThe UUID of the characteristic descriptor.
Returns
A GATT_BATTERY_CLIENT_READ_DESCRIPTOR_CFM message will be sent to the registered application Task.

◆ GattBatteryClientSetNotificationEnableRequest()

void GattBatteryClientSetNotificationEnableRequest ( const GBASC *  battery_client,
bool  notifications_enable 
)

This API is used to write the client characteristic configuration on a remote device, to enable notifications with the server. An error will be returned if the server does not support notifications.

Parameters
battery_clientThe client instance that was passed into the GattBatteryClientInit API.
notifications_enableSet to TRUE to enable notifications on the server, FALSE to disable them.
Returns
A GATT_BATTERY_CLIENT_SET_NOTIFICATION_ENABLE_CFM message will be sent to the registered application Task.
