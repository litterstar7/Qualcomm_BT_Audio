Detailed Description

Header file for the GATT AMS client library.

This file provides documentation for the GATT AMS client library API (library name: gatt_ams_client).

◆ GattAmsAddReadyStateObserver()

bool GattAmsAddReadyStateObserver ( gatt_ams_ready_state_observer_t  observer)

The function passed will be called when the ready state of a client instance changes. Ready means a client instance has just been fully initialised. Non-ready means a client instance is going to be destroyed immediately after this call.

Parameters
observerThe function called when the ready state changes.
Returns
TRUE when the observer has been successfully added, FALSE otherwise.

◆ GattAmsInit()

bool GattAmsInit ( Task  task,
uint16  cid,
uint16  start_handle,
uint16  end_handle,
void *  ams_data_dynamic,
void *  ams_data_const 
)

After the VM application has used the GATT manager library to establish a connection to a discovered BLE device in the Client role, it can discover any supported services in which it has an interest. It should then register with the relevant client service library (passing the relevant CID and handles to the service). For the AMS client it will use this API. The GATT manager will then route notifications and indications to the correct instance of the client service library for the CID.

Parameters
taskThe Task that will receive the GATT_AMS_CLIENT_INIT_CFM message.
amsA valid area of memory that the service library can use.
cidThe connection ID.
start_handleThe start handle of the AMS client instance.
end_handleThe end handle of the AMS client instance.
Returns
The status result of calling the API.
