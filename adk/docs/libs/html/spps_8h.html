Detailed Description

Header file for the Serial Port Profile Server.

This library implements the Serial Port Profile (SPP) specification for a server side connection (Device B, Acceptor). SPP can be used to emulate a serial port connection using RFCOMM between two peer devices.

The library exposes a functional downstream API and an upstream message based API.

void spps_set_sdp_service_record ( const uint8 *  record,
uint16  record_size 
)

Specify a record to be used for the SPP service in service discovery

The SPP Server has a default record which will otherwise be used.

A NULL parameter will ensure that the default record is used.

  • This function must be called before SppStartService
  • The RFCOMM port in the record will be updated to match the rfcomm port that is registered by the service
  • The record supplied MAY be const data. It is copied when used.
  • The record supplied MUST NOT be from local memory, or memory that is freed while the service is running
Parameters
[in]recordPointer to the service record to be used. See the notes
record_sizeThe size of the service record in bytes.
