Detailed Description

Header file for the GATT root key client library.

This file provides documentation for the GATT role selection client library API (library name: gatt_role_selection_client).

The design intention is that the role selection service is fast for the normal use case. The normal case is to command the peer to change to primary or secondary role.

If notifications are required then the required descriptor handle will be discovered when required.

bool GattRoleSelectionClientDestroy ( GATT_ROLE_SELECTION_CLIENT *  instance)

When a GATT connection is removed, the application must remove all client service instances that were associated with the connection (using the CID value). This is the clean up routine as a result of calling the GattRoleSelectionClientInit API. That is, the GattRoleSelectionClientInit API is called when a connection is made, and the GattRoleSelectionClientDestroy is called when the connection is removed.

◆ GattRoleSelectionClientInit()

bool GattRoleSelectionClientInit ( GATT_ROLE_SELECTION_CLIENT *  instance,
Task  app_task,
gatt_role_selection_handles_t *  cached_handles,
uint16  cid,
uint16  start_handle,
uint16  end_handle 
)

After the VM application has used the GATT manager library to establish a connection to a discovered BLE device in the Client role, it can discover any supported services in which it has an interest. It should then register with the relevant client service library (passing the relevant CID and handles to the service). For the role_selection client it will use this API. The GATT manager will then route notifications and indications to the correct instance of the client service library for the CID. If the characteristic handles are already known they can be passed to the library with the parameter 'cached_handles'. This will cause the libray to not retrieve the handles from the server. If the handles are not known set cached_handles to NULL.

Parameters
instanceA valid area of memory that the service library can use.
app_taskThe Task that will receive the messages sent from this role_selection client library.
cached_handlesHandles of characteristics if already known. Otherwise NULL.
cidThe connection ID.
start_handleThe start handle of the role_selection client instance.
end_handleThe end handle of the role_selection client instance.
Returns
TRUE if successful, FALSE otherwise
