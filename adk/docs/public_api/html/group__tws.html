#include <adk/src/topologies/tws/common/core/tws_topology.h>

Definition of messages that TWS Topology can send to clients.

Enumerator
TWS_TOPOLOGY_INIT_CFM 

Confirmation that the TWS Topology module has initialised, sent once TwsTopology_Init() has completed

TWS_TOPOLOGY_START_CFM 

Confirmation that TWS Topology has started, sent in response to TwsTopology_Start()

TWS_TOPOLOGY_STOP_CFM 

Confirmation that TWS Topology has stopped, sent in response to TwsTopology_Stop()

TWS_TOPOLOGY_ROLE_CHANGED_IND 

Indication to clients that the Earbud role has changed.

TWS_TOPOLOGY_HANDSET_DISCONNECTED_IND 

Indication to clients that handset have been disconnected.

◆ TwsTopology_EndDfu()

void TwsTopology_EndDfu ( void  )

#include <adk/src/topologies/tws/common/core/tws_topology.h>

End DFU.

Tell the topology that DFU has completed, regardless of whether it was successful.

This indication is significant when the DFU ends as in-case. i.e. If indicated when in-case then the links can be dropped & eventually the main role is given up (as expected when in-case).
